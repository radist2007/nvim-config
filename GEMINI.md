# Нотатки по конфігурації Neovim

Цей файл містить ключову інформацію про поточну конфігурацію Neovim та історію її налаштування для майбутніх сесій.

## Стан системи

- **Neovim:** Встановлено версію **v0.12.0-dev**. Важливо: Neovim було скомпільовано та встановлено з вихідного коду (`git clone ... && make install`), оскільки стандартні методи (PPA, deb) виявилися проблематичними.
- **Конфігурація:** Використовується **LazyVim** з кастомними налаштуваннями в `~/.config/nvim/lua/plugins/`.
- **Ключовий плагін:** Налаштовано `avante.nvim` для інтеграції з **Gemini**.

## Історія важливих змін (Жовтень 2025)

Проведено велике оновлення з Neovim 0.11.0 до 0.12.0-dev, що було викликано оновленням LazyVim. Це спричинило низку проблем, які були вирішені наступним чином:

1.  **Оновлення плагінів:** Виконано `:Lazy update` та `:Lazy clean` для синхронізації всіх плагінів з новою версією Neovim та виправлення численних Lua-помилок.
2.  **Виправлення шляхів:** Оновлено шляхи до плагінів `mason.nvim` та `mason-lspconfig.nvim` на нову адресу `mason-org/...` у відповідних файлах конфігурації.
3.  **Конфігурація Tree-sitter:** Видалено застарілу секцію `config` з файлу `lua/plugins/treesitter.lua`, оскільки API плагіна змінився. Для синхронізації парсерів виконано `:TSUpdate`.
4.  **Конфігурація `avante.nvim`:** Виправлено кілька синтаксичних помилок та оновлено структуру конфігурації для відповідності новій версії плагіна (перенесено налаштування Gemini в секцію `providers`).
5.  **Зміни в UI:** Зафіксовано, що зміна вигляду командного рядка (з центрального на стандартний нижній) є **очікуваною поведінкою**, оскільки LazyVim замінив плагін `dressing.nvim`.
6.  **Застарілі функції:** Виправлено попередження про застарілі функції `vim.lsp.set_log_level` та `LazyVim.ui.foldexpr`.
